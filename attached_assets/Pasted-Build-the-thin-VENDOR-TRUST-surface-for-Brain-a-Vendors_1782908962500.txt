TRUST MODEL (the spine): trust is a status a vendor holds, granted by the user but informed by
Brain's evidence, and it gates automation eligibility.
- Statuses: "new" (0-2 payments, always reviewed, never auto-clearable), "known" (history
  building, still reviewed, not allowlist-eligible), "trusted" (user-vetted, the ONLY status
  eligible for a rule allowlist / auto-clear), "under_review" (trust suspended after a flag,
  treated like new until resolved).
- HARD RULE: a bank-detail / payee-account change on a TRUSTED vendor auto-drops it to
  "under_review". Trust is bound to the vendor AND the account paid; change the account and
  trust does not carry over. (Same guard as policy/ap.fraud.v2.)
- Brain computes eligibility and SUGGESTS trust but NEVER auto-grants. Granting is always an
  explicit human action.

DATA MODEL:
type TrustStatus = "new" | "known" | "trusted" | "under_review";
interface VendorFlag { kind: "bank_detail_change"|"amount_anomaly"|"reported_problem"|"possible_duplicate"; label: string; raisedAtLabel: string; priorAccountLast4?: string; newAccountLast4?: string; }
interface Vendor {
  id: string; name: string; category: string; trustStatus: TrustStatus;
  payeeAccountLast4: string;
  history: { paymentCount: number; totalPaid: number; firstPaidLabel: string; lastPaidLabel: string; avgAmount: number; flagCount: number; };
  flags: VendorFlag[];
  trustGrantedLabel?: string;      // "You trusted this vendor Jun 12 · 7 payments since"
  wasTrustedLabel?: string;        // for under_review: "was trusted since May 2"
  eligibleForTrust: boolean;       // Brain's signal → drives the suggestion (known only)
  eligibilityEvidence?: FactRow[]; // the facts behind the suggestion
  ruleIds: string[];               // rules whose allowlist includes this vendor
}
