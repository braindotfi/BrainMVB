Fix wiki.question audit records showing an identical, generic title ("Assistant asked a question") for every occurrence, instead of surfacing what was actually asked.
Background: brain-core's wiki.question event already includes the real question text in inputs.question (this shipped in an earlier fix and is documented in the api-surface.brainmvb.json contract under audit_event_contract.event_types.assistant_activity.action_guarantees["wiki.question"]). The client currently ignores it — ACTION_MAP["wiki.question"].summary in client/src/lib/brainAudit.ts is a hardcoded string that never reads the event payload, so every wiki.question record renders with the same title regardless of what was asked. Two records six minutes apart are currently visually indistinguishable.
Changes needed:

In client/src/lib/brainAudit.ts, update the wiki.question entry in ACTION_MAP so summary(e) reads e.inputs.question (a string) and uses it as the title, instead of the fixed "Assistant asked a question" string. Fall back to the current generic text only if inputs.question is missing or not a string (defensive, shouldn't normally happen given the contract guarantee).
For the Audit Log list card and Inbox card (wherever this record type renders as a single-line title), truncate the question to a reasonable card-friendly length (e.g. ~60-80 characters with an ellipsis) so long questions don't break the row layout.
In AuditRecordPopup.tsx, show the full, untruncated question text — either as the title if it fits, or as a line in the Decision Lifecycle section beneath the truncated title if it's long. Don't truncate it in the popup the way the list card does.
Add an actor line to AuditRecordPopup.tsx for this record type (and any other record type currently missing one) using the actor resolution work that already landed (humanReadableActor / the actor_ref lookup via the BFF) — e.g. "Asked on behalf of {actor}" or similar, omitted per the existing honest-omission behavior when no resolvable actor is available. Check whether this line is missing more broadly across the popup, not just for wiki.question — if other record types also lack an actor line where one would be meaningful, add it consistently rather than only for this one type.

Scope: this touches brainAudit.ts (the wiki.question summary function) and AuditRecordPopup.tsx (question text display + actor line). No other event types, pages, or classification logic should change — this is a content-surfacing fix, not another taxonomy change.
